# 06 — Roadmap et méthode de travail

## Principe général

Le projet Les Voyages de Cha doit avancer par petites briques.

Chaque brique doit être limitée, vérifiable et enregistrée en Git avant de passer à la suivante.

Objectif : éviter la dispersion, protéger les informations importantes et permettre de revenir en arrière si nécessaire.

## Règle de travail

À chaque étape :

1. vérifier le dossier exact du projet ;
2. modifier une seule partie ;
3. tester si nécessaire ;
4. enregistrer en Git ;
5. pousser sur GitHub ;
6. seulement ensuite passer à la suite.

## Règle terminal

Toute commande doit être complète.

Chaque bloc de commande doit commencer par :

cd /home/sofia/Les-voyages-de-Cha
pwd
ls

Quand le site doit être visible, terminer par :

npm run dev

Puis ouvrir :

http://localhost:3000

Si le terminal indique un autre port, ouvrir celui affiché.

## V1 — Socle sécurisé

Objectif : poser une base propre, documentée et sauvegardée.

À faire :

- créer le dossier projet ;
- créer la base Next.js ;
- créer le README ;
- créer la documentation métier ;
- initialiser Git local ;
- pousser sur GitHub ;
- sécuriser la direction artistique ;
- conserver les décisions importantes dans docs.

## V2 — Vitrine immersive

Objectif : créer une page d'accueil qui donne envie de voyager.

À faire :

- harmoniser les couleurs selon la direction artistique ;
- renforcer le beige dominant ;
- garder le bleu profond comme accent ;
- intégrer de belles images de voyage ;
- intégrer le logo proprement ;
- créer une hero plus émotionnelle ;
- créer les sections À propos, Offres, Destinations, Avis et Contact ;
- ajouter le bouton Calendly ;
- ajouter la règle de non-réservation de manière élégante.

## V3 — Pages publiques

Pages prévues :

- Destinations ;

- Avis ;

Objectif : construire le début du back-office métier.

Modules prioritaires :

- CRM / portefeuille clients ;

- base destinations ;
- captures rapides ;
- templates mails ;

- avis clients ;
- finances simples ;
- traceur de temps ;

## V5 — Automatisations

Objectif : réduire les tâches répétitives.

Automatisations possibles :

- questionnaire vers fiche client ;
- rendez-vous Calendly rattaché au prospect ;
- relance questionnaire ;
- relance devis ;

- demande d'avis ;
- création de tâches automatiques.

- ambiance bord de mer ;
- chaleur ;
- douceur ;

- émotion ;
- humain ;
- premium accessible ;
- jamais froid ;
- jamais corporate ;
