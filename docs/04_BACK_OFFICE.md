# 04 — Bureau de Cha / back-office

## Objectif général

Le Bureau de Cha est le QG métier de l'activité Les Voyages de Cha.

Il doit permettre à Cha de piloter son activité sans chercher ses informations dans dix endroits différents.

Promesse interne :

Tout capter. Tout relier. Tout retrouver. Tout réutiliser.

## Principe d'organisation

Le Bureau de Cha doit fonctionner comme une base centrale, proche d'une logique Notion ou ClickUp, avec des onglets métier clairs.

Il doit regrouper :

- clients ;
- prospects ;
- dossiers voyage ;
- destinations ;
- liens et inspirations ;
- cartes interactives ;
- carnets de voyage ;
- questionnaires ;
- mails et templates ;
- marketing ;
- réseaux sociaux ;
- visuels ;
- avis clients ;
- finances ;
- administratif ;
- conformité ;
- traceur de temps ;
- focus et énergie.

## Dossiers voyage

Chaque voyage est un dossier projet.

Un dossier voyage doit contenir :

- client lié ;
- destination ;
- dates ;
- durée ;
- nombre de voyageurs ;
- formule choisie ;
- budget cible ;
- contraintes ;
- rythme souhaité ;
- livrables prévus ;
- carte interactive oui/non ;
- recommandations ;
- liens ;
- contacts ;
- notes internes ;
- statut ;
- temps passé ;
- retour client.

Bloc obligatoire dans chaque dossier :

Cha ne réalise aucune réservation. Les recommandations, liens et contacts sont transmis au client, qui effectue lui-même ses réservations auprès des prestataires.

## Traceur de temps

Un bouton simple doit permettre de mesurer le temps passé sur :

- un client ;
- un dossier voyage ;
- un type de tâche ;
- une destination ;
- une prestation.

Types de tâches possibles :

- qualification client ;
- analyse questionnaire ;
- recherche destination ;
- recherche hébergement ;
- recherche activités ;
- création itinéraire ;
- carte interactive ;
- rédaction carnet de voyage ;
- échanges client ;
- devis / facture ;
- relance ;
- marketing ;
- administratif.

Phrase interne :

Je mesure mon temps pour protéger ma rentabilité, pas pour me fliquer.
